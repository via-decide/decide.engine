# ViaDecide UX Audit — 2026-03-07

Viewport testing of `index.html` across mobile, tablet, desktop, and TV breakpoints.

---

## Mobile (375 x 812)

### Touch-target violations (< 44 px minimum)
| Element | Actual size | Required |
|---------|------------|----------|
| `.nav-cta` (TRY ONDC) | 108 x 30 | 44 x 44 |
| `.orb-chip` (category pills) | ~70-104 x 36 | 44 x 44 |
| `.build-card-cta` (Open Tool) | 101 x 30 | 44 x 44 |
| `.build-search-btn` (Explore) | 317 x 35 | 44 x 44 |
| Footer links (Contact, Privacy, Terms, About) | ~31-53 x 19 | 44 x 44 |

**Impact:** Footer links are especially problematic — only 19 px tall, making them nearly impossible to tap accurately on mobile.

### Other mobile issues
- Hero section has excessive blank space above the tagline pill before scrolling into view.
- "SCROLL TO EXPLORE" hint is positioned below the fold; users may not discover scrollable content.

---

## Tablet (768 x 1024)

### Right-edge clipping
| Element | Issue |
|---------|-------|
| `.nav-cta` (TRY ONDC) | Button right edge at 725 px — text truncated to "TRY" only |
| "Our Philosophy" paragraph | Paragraph `scrollWidth` extends to 1081 px — text runs off-screen |
| "Popular tools" subtitle | "Dedicated subpages launching soon..." clipped on right |
| "Book a Session" button (Personal Brand section) | "Book" text cut off at viewport edge |

**Root cause:** Several sections use fixed or insufficiently constrained widths that don't adapt at the 768 px breakpoint. The nav likely needs `max-width: 100%; overflow: hidden` or responsive padding.

### Other tablet issues
- Tool cards in the "Popular tools" grid — right-column cards (e.g. "App Generator") have description text truncated.
- `.thread-empty-sub` paragraph overflows to 1081 px (should be contained within viewport).

---

## Desktop (1280 x 800)

Desktop layout is generally solid. Minor observations:

- **No critical bugs found** at this breakpoint.
- Card grid (3-column) renders cleanly.
- Nav items are legible and well-spaced.
- Footer links still lack `.html` extensions (router-dependent, see Cross-viewport below).

---

## TV (1920 x 1080)

### Readability at distance
| Element | Font size | Issue |
|---------|-----------|-------|
| Nav category links | 11.52 px | Too small for TV viewing distance (3-10 ft) |
| Nav CTA | 12.8 px | Borderline readable |

### Layout scaling
- `<main>` has no `max-width` — content stretches to 1917 px, making cards and text blocks very spread out.
- Hero search bar is only 552 px wide on a 1920 px viewport — looks disproportionately small.
- Card descriptions become hard to scan because cards are widely spaced with narrow text columns inside them.

**Recommendation:** Add a `max-width: 1440px` (or similar) on the main content container with `margin: 0 auto` to keep content readable on ultrawide/TV screens.

## Cross-viewport issues

### Footer links missing `.html` extensions
All internal footer links use bare slug paths:
```
/contact   ->  should be  /contact.html
/privacy   ->  should be  /privacy.html
/terms     ->  should be  /terms.html
/founder   ->  should be  /founder.html
```
These rely on `VDRouter.go()` intercepting clicks. If the router isn't loaded or the link is opened in a new tab, they will 404 on the static server (Python `http.server`, direct Vercel hit without rewrite).

**Note:** PR #13 (merged) fixed `routesMap` entries but these footer `href` attributes in `index.html` still use extensionless paths.

### Internal link without `.html`
- `/printbydd-store` (Open Store link) — also missing extension.

### Modal iframe error on navigation
- Clicking certain tool cards triggers a modal with "Can't load in iframe" error (e.g., Sales Dashboard) — expected behavior for external origins, but the error UX could be improved with a friendlier message or direct redirect.
