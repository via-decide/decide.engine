# Debug: Mobile Google result opens `404 DEPLOYMENT_NOT_FOUND`

## What this error means
`DEPLOYMENT_NOT_FOUND` is generated by Vercel edge when a hostname/path resolves to a deployment that no longer exists (or domain alias points to the wrong Vercel project/deployment).

This is usually **infrastructure mapping**, not page-level HTML code.

## Most likely causes
1. Domain alias mismatch in Vercel (custom domain mapped to wrong/deleted deployment)
2. Cloudflare DNS/proxy forwarding to stale target
3. Google cached old URL (legacy path or old deployment URL)
4. No valid sitemap/canonical refresh, so stale URLs remain indexed longer

## What was fixed in this repo
- Corrected robots sitemap URL to the live domain.
- Added a concrete `sitemap.xml` for core stable routes to speed re-indexing of valid pages.

## Production checks (do in dashboard)
1. Vercel → Project → Domains
   - Ensure `viadecide.com` and `www.viadecide.com` are attached to the active production project.
2. Cloudflare DNS
   - Avoid duplicate/old records for apex/www.
   - If proxied, verify origin target is current Vercel project.
3. Remove stale aliases/deployments
   - Any old `*.vercel.app` aliases should be removed if no longer valid.
4. Google Search Console
   - Inspect failing URL.
   - Request reindex after deployment.
   - Submit updated `https://www.viadecide.com/sitemap.xml`.

## Quick verification commands
- `dig +short viadecide.com A`
- `dig +short www.viadecide.com CNAME`
- `curl -I https://www.viadecide.com/`
- `curl -I "https://www.google.com/search?q=viadecide"` (optional reference)

If these are clean and issue persists only on some mobile networks, it is often DNS/cache propagation; usually clears after alias + sitemap corrections and reindex.
